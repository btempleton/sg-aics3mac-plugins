Detailed Description

This suite provides access to low-level user-interaction events, such as mouse-up, mouse-down, and mouse-move.

To work with high-level events, such as the value of an item changing, use the ADMNotifierSuite2.

You can modify or replace the default behavior of a dialog or dialog item by providing your own handler, in the form of a tracker callback procedure, ADMItemTrackProc, associated with the dialog or item. See ADMDialogSuite10::SetTrackProc() and ADMItemSuite9::SetTrackProc().

The behavior of list entries (ADMEntryRef or ADMListEntryRef) is handled by the tracker callback (ADMEntryTrackProc) of the containing list (ADMListRef or ADMHierarchyListRef). See ADMListSuite4::SetTrackProc() and ADMHierarchyListSuite5::SetTrackProc().

To register an interest in specific low-level events, set the event mask for the dialog, item, or list. See, for example, ADMItemSuite9::SetMask(). Only those events of interest trigger your tracking procedure.

The tracker object, ADMTrackerRef, provides the event context. It stores input state information, and provides a snapshot of input activity that is in progress when the tracker is activated.

To invoke the default behavior of a dialog, item, or list, call that object's default tracker handler from within your handler; for example, ADMItemSuite9::DefaultTrack().

  • Acquire this suite using SPBasicSuite::AcquireSuite() with the constants kADMTrackerSuite and kADMTrackerSuiteVersion2.
