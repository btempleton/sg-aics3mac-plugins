00059 /*      INTERNAL DOCS
00060  *      PICA makes callbacks into the host through the host procs. The host
00061  *      procs are filled in by the host and passed to Sweet Pea at SPInit().
00062  *
00063  *      hostData - data that is given back to each host proc when Sweet Pea
00064  *              calls it. Sweet Pea does nothing with it itself.
00065  *
00066  *      extAllocate - implementation of the Block Suite's AllocateBlock() routine.
00067  *              It is identical to ANSI C malloc(). It returns a pointer to the
00068  *              beginning of the allocated block or NULL.
00069  *
00070  *      extFree - implementation of the Block Suite's FreeBlock() routine. It is
00071  *              identical to ANSI C free(). Note that you can pass it NULL.
00072  *
00073  *      extReallocate - implementation of the Block Suite's ReallocateBlock()
00074  *              routine. It is identical to ANSI C realloc(). It returns a pointer
00075  *              to the resized block or NULL. Note that you can pass it NULL or a
00076  *              newSize of 0.
00077  *
00078  *      intAllocate, intFree, intReallocate - routines used by Sweet Pea for
00079  *              its own memory needs. You may want to allocate blocks differently
00080  *              with plug-ins and Sweet Pea. Plug-ins are unbounded in their memory
00081  *              needs, while Sweet Pea's memory usage can be approximated.
00082  *
00083  *      startupNotify - called as each plug-in is started up. This is intended
00084  *              as a way to tell the user what's happening during start up.
00085  *              Note that plug-ins may start up at any time, not just during
00086  *              SPStartupPlugins().
00087  *
00088  *      shutdownNotify - called as each plug-in is shut down. Also intended as
00089  *              a way to let users know what's going on.
00090  *
00091  *      assertTrap - called when a fatal assert is triggered. Sweet Pea does
00092  *              not expect execution to continue after an assert.
00093  *
00094  *      throwTrap - called when an internal error is thrown. This can be used
00095  *              during debugging to catch errors as they happen. It should return
00096  *              to allow Sweet Pea to handle the error.
00097  *
00098  *
00099  *      To aid in getting Sweet Pea up and running quickly, you can set any of
00100  *      these to NULL and Sweet Pea will use a default implementation. However:
00101  *      you cannot mix your implementations of the memory routines with
00102  *      Sweet Pea's defaults.
00103  *
00104  *
00105  *      The string pool functions replace the default routines used internally
00106  *      and exported by the Strings suite.  Because they are exported, the behaviors
00107  *      listed below should be followed.
00108  *
00109  *      allocateStringPool - creates a new string pool instance. The host app and
00110  *              Sweet Pea have a string pool which can be used by a plug-in, or a plug-in
00111  *              can create its own.  See the notes in SPStrngs.h on how the pool is
00112  *              implemented.
00113  *              The function should return kSPNoError if the pool is allocated successfully
00114  *              or kSPOutOfMemoryError if allocation fails.
00115  *
00116  *      freeStringPool - disposes of the string pool and any associated memory.  The
00117  *              function should return kSPNoError
00118  *
00119  *      makeWString - the string pool keeps a list of added strings. When a new string is
00120  *              added with MakeWString(), the routine checks to see if it is already in the
00121  *              pool.  If so, the address of the string instance in the pool is returned.  If
00122  *              not, it will add it to the pool and return the address of the newly
00123  *              created string instance.  The behavior is:
00124  *
00125  *                      if ( string == NULL )
00126  *                                      *wString = NULL;
00127  *                                      returns kSPNoError;
00128  *                      else if ( string in string pool )
00129  *                                      *wString = found string;
00130  *                                      returns kSPNoError;
00131  *                      else add string
00132  *                              if successful
00133  *                                      *wString = new string;
00134  *                                      returns kSPNoError;
00135  *                              else
00136  *                                      *wString = nil
00137  *                                      returns kSPOutOfMemoryError
00138  *
00139  *      appStringPool - if the host application has already allocated a string pool to use,
00140  *              it's reference should be passed here.  If this value is NULL, Sweet Pea will
00141  *              allocate the pool when initialized and dispose of it at termination.
00142  *
00143  *      filterEvent - a function called for each event allowing the host to cancel it.
00144  *              The event type is indicative of what the filter is to do.  A  file validation
00145  *              is called before a directory entry is added to the file list (kAddFile).
00146  *              A plug-in validation before a file is checked for PiPL information (kAddPlugin);
00147  *              the host might examine the file name/type to determine whether it should be added.
00148  *              For these 'add' events the return value is TRUE if the item should be skipped
00149  *              or FALSE if should be should be added. The default filter proc, used (if NULL)
00150  *              is passed, will skip files/folders in ( ).
00151  *              The other event is kSuitesAvailable.  It is called when the last suite adding
00152  *              plug-in (as determined by available PiPL information) has been added.  This is
00153  *              a point at which the host can cancel the startup process; for instance, if the host
00154  *              requires a suite from a plug-in, this is the time to check for it.  If the
00155  *              host returns TRUE, the startup process continues.  If it returns FALSE, the
00156  *              plug-in startup is canceled and the host would likely terminate or startup in
00157  *              an alternate manner.
00158  *
00159  *      overrideAddPlugins - if supplied, SP will call the host to create the runtime
00160  *              plug-in list.  This occurs at SPStartupPlugins().  The function takes no parameters
00161  *              as it is up to the host to determine how to do this.  For instance, the host can do
00162  *              this from cached data or, as SP would, from the file list.  A returned error will
00163  *              stop the plug-in startup process.
00164  *
00165  *      overrideStartup - a function called for each SP2 plug-in before it is sent the
00166  *              startup message.  If the host returns FALSE, SP will startup the plug-in normal.
00167  *              If the host returns true, it is assumed that the host has handled the startup
00168  *              for the plug-in, so SP will not do anything for the plug-in.  This is intended
00169  *              to be used with a plug-in caching scheme.
00170  *              The host would be responsible, for instance, for defining the cacheable
00171  *              information in the PiPL, adding it when the callback is made, and later issuing
00172  *              a startup message when the plug-in is actually needed (e.g. when a menu item
00173  *              is selected.)  Two notes: don't forget to SetPluginStarted(), and make sure
00174  *              to use a string pooled char* to kSPInterfaceCaller and kSPInterfaceStartupSelector.
00175  *
00176  *      resolveLink - Windows only.  If the search for plug-ins is to recurse sub-folders,
00177  *              the host needs to supply this routine.  When a .lnk file is encountered, the
00178  *              resolveLink host callback function will be called and should return a resolved path.
00179  *              This is a host callback due to OLE issues such as initialization, which the SP
00180  *              library does not currently handle.  If it returns an error code, the result will
00181  *              be ignored.
00182  *
00183  *      getPluginAccess - Allows the host to set the plug-in access information.  This would
00184  *              be used if, for instance, the host kept its own plug-in list (ala, Photoshop), but
00185  *              still needed these to be compatible with SPPlugins (e.g. whose accesses are used by ADM)
00186  *
00187  *      memoryIsCritical - Mac only.  Allows the host to indicate that memory is in a critical state
00188  *              (really low, but can't be purged because you are, say, shutdown.)
00189  *              If so and the plug-in load target heap is the app heap, when a plug-in fails to load
00190  *              SP will then try to load the plug-in into the system heap
00191  */
