Detailed Description

This suite allows you to create and destroy custom ADM dialogs, to configure and query dialogs, to invoke them, to manipulate their location and appearance, and to set and access their contents.

  • Acquire this suite using SPBasicSuite::AcquireSuite() with the constants kADMDialogSuite and kADMDialogSuiteVersion.

void ADMAPI(* ADMDialogSuite10::AbortTimer)(ADMDialogRef inDialog, ADMTimerRef inTimerID)

Cancels a scheduled action associated with a dialog.

It is used if an event specified by the inAbortMask of CreateTimer() occurs, or if you destroy the dialog before the timer expires.

Parameters:
inDialog The dialog object.
inTimerID The timer object containing the scheduled action.
Returns:
Nothing.

void ADMAPI(* ADMDialogSuite10::Activate)(ADMDialogRef inDialog, ADMBoolean inActivate)

Activates or deactivates a floating dialog.

The active dialog is the frontmost dialog, which has the input focus. When a modal dialog is active, no floating dialogs can become active.

Parameters:
inDialog The dialog object.
inActivate True to activate the dialog, false to deactivate it.
Returns:
Nothing.

ADMErr ADMAPI(* ADMDialogSuite10::BeginAdjustingFocusOrder)(ADMDialogRef inDialog)

Opens a sequence of calls to ADMItemSuite9::SetFirstFocus() and ADMItemSuite9::SetNextFocus(), which set the focus order (also called tab order) for a set of items in a dialog.

This is the order in which contained elements get input focus when the user presses the Tab key.

When the sequence is finished, call DoneAdjustingFocusOrder().

This code sequence must not be nested. Finish specifying the order in one dialog before proceeding to the next one.

Parameters:
inDialog The dialog object.
Note:
To rearrange the order of an existing focus list, use ADMItemSuite9::MoveFocusItem().
See also:
FirstItem(), LastItem()

void ADMAPI(* ADMDialogSuite10::Destroy)(ADMDialogRef inDialog)

Deletes a dialog object, calling its associated ADMDialogDestroyProc first.

Modal dialogs created with Modal() are destroyed automatically when they are dismissed. All dialogs are destroyed automatically on application shutdown. You must call this to destroy any non-modal dialogs, or modal dialogs created with Create(), if your plug-in shuts down before the application does (typically in response to a plug-in shutdown message) or when you are finished with them (typically in response to an ADM close-window message).

Note:
Must not be called twice for the same dialog.
Parameters:
inDialog The dialog object.
Returns:
Nothing.
