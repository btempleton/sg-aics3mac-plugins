Detailed Description

This suite provides access to high-level user-interaction events, such as the value of an item changing.

To work with low-level events, such as mouse-up, mouse-down, and mouse-move, use the ADMTrackerSuite2.

You can modify or replace the default behavior of a dialog or dialog item by providing your own handler, in the form of a notification callback procedure, ADMItemNotifyProc, associated with the dialog or item. See ADMDialogSuite10::SetNotifyProc() and ADMItemSuite9::SetNotifyProc().

The behavior of list entries (ADMEntryRef or ADMListEntryRef) is handled by the notifier callback (ADMEntryNotifyProc) of the containing list (ADMListRef or ADMHierarchyListRef). See ADMListSuite4::SetNotifyProc() and ADMHierarchyListSuite5::SetNotifyProc().

The notification object, ADMNotifierRef, provides the event context. Use the functions in this suite in your notification callback procedure to access information about the event that caused the notification.

To invoke the default behavior of a dialog, item, or list, call that object's default notification handler from within your handler; for example, ADMItemSuite9::DefaultNotify().

  • Acquire this suite using SPBasicSuite::AcquireSuite() with the constants kADMNotifierSuite and kADMNotifierSuiteVersion2.
