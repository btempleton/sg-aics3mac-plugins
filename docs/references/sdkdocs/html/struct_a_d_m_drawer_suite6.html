Detailed Description

This suite allows you to modify or replace the way dialog, items, and lists are drawn on the screen.

You can modify or replace the default drawing method of a dialog or dialog item by providing your own handler, in the form of a drawing callback procedure, ADMItemDrawProc, associated with the dialog or item. See ADMDialogSuite10::SetDrawProc() and ADMItemSuite9::SetDrawProc().

The drawing of list entries (ADMEntryRef or ADMListEntryRef) is handled by the drawing callback (ADMEntryDrawProc) of the containing list (ADMListRef or ADMHierarchyListRef). See ADMListSuite4::SetDrawProc() and ADMHierarchyListSuite5::SetDrawProc().

The drawer object, ADMDrawerRef, provides access to the platform-independent imaging functions in this suite, which include basic imaging, such as DrawLine(), and text handling. The suite makes use of the streamlined ADM color and font style constants, and provides functions that simplify the implementation of many standard dialog elements, such as DrawIcon() and DrawUpArrow().

All drawing is done in coordinates local to the object being drawn, whose origin is, by default, the top-left corner of the bounding rectangle. The drawer has an update region, and draws only within that region. You can further specify a clipping path for a drawer; pixels that lie outside the clip region are not drawn.

To invoke the default drawing method of a dialog, item, or list, call that object's default drawing handler from within your handler; for example, ADMItemSuite9::DefaultDraw().

  • Acquire this suite using SPBasicSuite::AcquireSuite() with the constants kADMDrawerSuite and kADMDrawerSuiteVersion6.

ADMDrawerRef ADMAPI(* ADMDrawerSuite6::Create)(ADMPortRef inPortRef, const ADMRect *inBoundsRect, ADMFont inFont)

Creates a drawing object.

ADM does this automatically before calling a drawing procedure, and passes the object to the procedure. If you create a drawing object, you must free it when it is no longer needed, using Destroy().

Parameters:
inPortRef The window drawing port for the object to be drawn. Obtain the window (with, for example, ADMDialogSuite10::GetWindowRef()) then use it to object the port with GetADMWindowPort().
inBoundsRect Optional. The bounding box of the update region. All drawing is within this region, which can be further restricted with a clipping path. Default is the bounding box of the object being drawn. Coordinates are in pixels, relative to the origin (top left) of the object being drawn.
inFont The font style to use for drawing text, a font-style constant.
Returns:
The new drawing object.
